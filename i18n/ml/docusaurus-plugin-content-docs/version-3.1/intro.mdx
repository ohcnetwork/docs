# കെയർ ഡോക്യുമെന്റേഷൻ

കെയർ ഡോക്യുമെന്റേഷൻ നാല് പാളികളായി ക്രമീകരിച്ചിരിക്കുന്നു. മാനസിക മാതൃകകൾ നിർമ്മിക്കാൻ **ആശയങ്ങൾ** ഉപയോഗിക്കുക, ഉൽപ്പന്നത്തിൽ എന്തെങ്കിലും ചെയ്യണമെങ്കിൽ **ഫ്ലോകൾ** പിന്തുടരുക, സാങ്കേതിക വിശദാംശങ്ങൾക്ക് **റഫറൻസുകൾ** ഉപയോഗിക്കുക, ഒരു ഡിപ്ലോയ്മെന്റ് ഫ്ലോകളെ യഥാർത്ഥ സാഹചര്യവുമായി ബന്ധിപ്പിക്കുമ്പോൾ **പ്ലേബുക്കുകൾ** ഉപയോഗിക്കുക.

## ആശയങ്ങൾ

**ആശയങ്ങൾ ഇതിന് ഉത്തരം നൽകുന്നു: _എന്താണ്…?_**

ലളിത ഭാഷയിൽ നിർവചനങ്ങളും പ്ലാറ്റ്ഫോം അടിസ്ഥാനഘടകങ്ങളും — ഉദാഹരണത്തിന്, [രോഗി എന്താണ്](./concepts/clinical/patient.mdx). ആശയങ്ങൾ കോർ കെയർ റിലീസിനൊപ്പം പതിപ്പിടപ്പെടുത്തിയിട്ടുണ്ട്, എല്ലാ ഡിപ്ലോയ്മെന്റുകളിലും ബാധകമാണ്.

→ [ആശയങ്ങൾ](./concepts/clinical/patient.mdx) ബ്രൗസ് ചെയ്യുക

## ഫ്ലോകൾ

**ഫ്ലോകൾ ഇതിന് ഉത്തരം നൽകുന്നു: _എങ്ങനെ ചെയ്യാം…?_**

കെയറിലെ ജോലികൾക്കുള്ള ഘട്ടം ഘട്ടമായുള്ള ഗൈഡുകൾ — ഉദാഹരണത്തിന്, [രോഗിയെ എങ്ങനെ സൃഷ്ടിക്കാം](./flows/clinical/create-patient.mdx). ഫ്ലോകൾ ഉൽപ്പന്ന UI-യിലും API-യിലും സാധാരണ പാത വിവരിക്കുന്നു; കുറിച്ചില്ലെങ്കിൽ ഡിപ്ലോയ്മെന്റ്-നിരപേക്ഷമാണ്.

→ [ഫ്ലോകൾ](./flows/clinical/create-patient.mdx) ബ്രൗസ് ചെയ്യുക

## റഫറൻസുകൾ

**റഫറൻസുകൾ ഇതിന് ഉത്തരം നൽകുന്നു: _എങ്ങനെയാണ് നിർമ്മിച്ചിരിക്കുന്നത്?_**

ഡെവലപ്പർമാർക്കും ഇന്റഗ്രേറ്റർമാർക്കുമുള്ള മൊഡ്യൂൾ-തല സാങ്കേതിക ഡോക്യുമെന്റേഷൻ — മോഡലുകൾ, ഫീൽഡുകൾ, ബന്ധങ്ങൾ, സേവ് പെരുമാറ്റം. ഉദാഹരണത്തിന്, [രോഗി റഫറൻസ്](./references/clinical/patient.mdx) `care.emr.models.patient`-ലേക്ക് മാപ്പ് ചെയ്യുന്നു.

→ [റഫറൻസുകൾ](./references/clinical/patient.mdx) ബ്രൗസ് ചെയ്യുക

## പ്ലേബുക്കുകൾ

**പ്ലേബുക്കുകൾ ഇതിന് ഉത്തരം നൽകുന്നു: _എന്റെ ഡിപ്ലോയ്മെന്റിൽ ഇത് എങ്ങനെ പ്രവർത്തിക്കും?_**

പ്ലേബുക്കുകൾ ഓരോ [ഡിപ്ലോയ്മെന്റ് പ്ലേബുക്ക്](/deployments/hmis/)-നും (HMIS മറ്റുള്ളവ) കീഴിൽ ഉണ്ട്. രജിസ്ട്രേഷൻ ഡെസ്ക്, എമർജൻസി ഇൻടേക്ക്, വാർഡ് വർക്ക്ഫ്ലോകൾ തുടങ്ങിയ konkret ഉപയോഗകേസുകൾക്കായി ഫ്ലോകൾ ബന്ധിപ്പിക്കുകയും എഡ്ജ് കേസുകൾ, പ്രാദേശിക കോൺഫിഗറേഷൻ, പ്രവർത്തന പ്രതീക്ഷകൾ രേഖപ്പെടുത്തുകയും ചെയ്യുന്നു.

→ [HMIS പ്ലേബുക്കുകൾ](/deployments/hmis/playbooks/outpatient-registration) ബ്രൗസ് ചെയ്യുക
