# രോഗിയെ സൃഷ്ടിക്കുക

കെയറിൽ **പുതിയ രോഗിയെ** രജിസ്റ്റർ ചെയ്യുന്നതിന്റെ ഘട്ടങ്ങൾ — തിരയൽ മുതൽ എൻകൗണ്ടറിന് തയ്യാറായ സേവ് ചെയ്ത രേഖ വരെ.

:::info മുൻവശം
രോഗി രേഖയിൽ എന്തുണ്ട്, ഐഡന്റിഫയറുകൾ എങ്ങനെ പ്രവർത്തിക്കുന്നു എന്ന് മനസ്സിലാക്കാൻ [രോഗി](../../concepts/clinical/patient.mdx) വായിക്കുക.
:::

## ആരംഭിക്കുന്നതിന് മുമ്പ്

- **രോഗി സൃഷ്ടിക്കൽ** അനുമതിയുള്ള റോൾ വേണം (സാധാരണയായി രജിസ്ട്രേഷൻ/ഫ്രണ്ട് ഡെസ്ക്).
- നിങ്ങളുടെ സൗകര്യത്തിന് ഏത് **ഐഡന്റിഫയറുകൾ** ആവശ്യമാണെന്ന് അറിയുക (MRN, ദേശീയ ID, ഫോൺ മാത്രം ഇൻടേക്ക് തുടങ്ങിയവ).
- ഡ്യൂപ്ലിക്കേറ്റ് പരിശോധന നടപ്പിലാക്കിയാൽ, കുറഞ്ഞത് രണ്ട് ഐഡന്റിഫയറുകൾ തയ്യാറാക്കുക (ഉദാ. പേര് + ജനനത്തീയതി + ഫോൺ).

## ഘട്ടങ്ങൾ

### 1. രോഗി തിരയൽ തുറക്കുക

സൗകര്യ ഡാഷ്‌ബോർഡിൽ **Patients** → **Search** (അല്ലെങ്കിൽ ഗ്ലോബൽ രോഗി തിരയൽ) ഉപയോഗിക്കുക.

ആദ്യം തിരയുക — ഡ്യൂപ്ലിക്കേറ്റ് സൃഷ്ടിക്കുന്നതിന് മുമ്പ് വ്യക്തി ഇതിനകം രജിസ്റ്റർ ചെയ്തിട്ടില്ലെന്ന് ഉറപ്പാക്കാൻ കെയർ പ്രതീക്ഷിക്കുന്നു.

### 2. രജിസ്ട്രേഷൻ ആരംഭിക്കുക

പൊരുത്തം കിട്ടിയില്ലെങ്കിൽ **Register new patient** തിരഞ്ഞെടുക്കുക.

രജിസ്ട്രേഷൻ ഫോം തുറക്കുന്നു. ആവശ്യമായ ഫീൽഡുകൾ നിങ്ങളുടെ സൗകര്യത്തിന്റെ ചോദ്യാവലി കോൺഫിഗറേഷനെ ആശ്രയിച്ചിരിക്കുന്നു; അടിസ്ഥാന സെറ്റ്:

| ഫീൽഡ് | കുറിപ്പുകൾ |
| --- | --- |
| പേര് | നിയമപരമായ പേര്; കോൺഫിഗർ ചെയ്താൽ ഒന്നിലധികം പേരുകൾ |
| ജനനത്തീയതി | ഡ്യൂപ്ലിക്കേറ്റ് കണ്ടെത്തലിന് |
| ലിംഗം | ക്ലിനിക്കൽ സന്ദർഭത്തിനുള്ള അഡ്മിനിസ്ട്രേറ്റീവ് ലിംഗം |
| ഫോൺ | OTP/ഫോളോ-അപ്പിന് പലപ്പോഴും ആവശ്യം |
| ഐഡന്റിഫയർ | സൗകര്യ MRN അല്ലെങ്കിൽ ദേശീയ ID |

### 3. ഡെമോഗ്രാഫിക്സ് പൂർത്തിയാക്കുക

പ്രോഗ്രാമിന് ആവശ്യമായ ഓപ്ഷണൽ ഫീൽഡുകൾ പൂരിപ്പിക്കുക — വിലാസം, അടിയന്തര ബന്ധം, രക്തഗ്രൂപ്പ്, ഫോട്ടോ തുടങ്ങിയവ.

ഡ്യൂപ്ലിക്കേറ്റ് കണ്ടെത്തൽ പ്രവർത്തിക്കുന്നുവെങ്കിൽ, സമാന രേഖയുണ്ടാകുമ്പോൾ മുന്നറിയിപ്പ് കാണാം. സേവ് ചെയ്യുന്നതിന് മുമ്പ് പരിഹരിക്കുക (മർജ്, കാരണം രേഖപ്പെടുത്തി ഓവർറൈഡ്, അല്ലെങ്കിൽ റദ്ദാക്കുക).

### 4. രേഖ സേവ് ചെയ്യുക

**Save** (അല്ലെങ്കിൽ **Register**) തിരഞ്ഞെടുക്കുക.

വിജയിച്ചാൽ:

- രോഗി ID, സൗകര്യ MRN (ഓട്ടോ-ജനറേറ്റ് ചെയ്താൽ) നൽകുന്നു.
- രോഗി പ്രൊഫൈൽ തുറക്കുന്നു.
- **എൻകൗണ്ടർ** ആരംഭിക്കാം അല്ലെങ്കിൽ ക്യൂവിലേക്ക് മടങ്ങാം.

## സൃഷ്ടിച്ചതിന് ശേഷം

| അടുത്ത ഘട്ടം | എപ്പോൾ |
| --- | --- |
| എൻകൗണ്ടർ ആരംഭിക്കുക | രോഗി ഡെസ്കിൽ സന്ദർശനത്തിന് *(ഫ്ലോ ഉടൻ)* |
| രജിസ്ട്രേഷൻ സ്ലിപ്പ് പ്രിന്റ് | സൗകര്യ നയം പേപ്പർ ടോക്കൺ ആവശ്യപ്പെടുന്നു |
| ദേശീയ ആരോഗ്യ ID ലിങ്ക് | ABDM മറ്റ് റെയിലുകൾ പ്രവർത്തനക്ഷമമാക്കിയിട്ടുണ്ട് |

## API തുല്യം

```http
POST /api/v1/patient/
Content-Type: application/json

{
  "name": "Jane Doe",
  "age": 34,
  "gender": "female",
  "phone_number": "+91XXXXXXXXXX"
}
```

കൃത്യമായ payload ആകൃതി നിങ്ങളുടെ കെയർ പതിപ്പും സൗകര്യ ചോദ്യാവലിയും ആശ്രയിച്ചിരിക്കുന്നു. നിങ്ങളുടെ ഇൻസ്റ്റൻസിന്റെ OpenAPI സ്കീമ ഉപയോഗിക്കുക.

## എഡ്ജ് കേസുകൾ

അജ്ഞാത ഐഡന്റിറ്റി, ID ഇല്ലാത്ത എമർജൻസി ഇൻടേക്ക്, ഡ്യൂപ്ലിക്കേറ്റ് മർജ് തുടങ്ങിയ ഡിപ്ലോയ്മെന്റ്-നിർദ്ദിഷ്ട കൈകാര്യം ചെയ്യൽ പ്ലേബുക്കുകളിൽ രേഖപ്പെടുത്തിയിട്ടുണ്ട്, ഇവിടെയല്ല.

→ HMIS രജിസ്ട്രേഷൻ ഡെസ്ക് സാഹചര്യങ്ങൾക്ക് [ഔട്ട്‌പേഷ്യന്റ് രജിസ്ട്രേഷൻ](/deployments/hmis/playbooks/outpatient-registration) കാണുക.
