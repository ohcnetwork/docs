# രോഗി

കെയറിൽ **രോഗി** എന്നത് നിങ്ങളുടെ സൗകര്യമോ പ്രോഗ്രാമോ വഴി പരിചരണം ലഭിക്കുന്ന വ്യക്തിയാണ്. ഇത് ദീർഘകാലാന്തര പരിചരണ രേഖയുടെ അടിത്തറയാണ് — ഓരോ എൻകൗണ്ടർ, നിരീക്ഷണം, ഓർഡർ, കെയർ പ്ലാൻ എന്നിവ രോഗി രേഖയിലേക്ക് ലിങ്ക് ചെയ്യപ്പെടുന്നു.

## ഇത് എന്താണ് പ്രതിനിധീകരിക്കുന്നത്

കെയറിന്റെ FHIR-അനുസൃത മോഡലിൽ, രോഗി **Patient** റിസോഴ്സിലേക്ക് മാപ്പ് ചെയ്യുന്നു. ഇതിൽ ഉൾപ്പെടുന്നത്:

- **ഐഡന്റിറ്റി** — പേര്, ജനനത്തീയതി, ലിംഗം, ഫോട്ടോ, സർക്കാർ അല്ലെങ്കിൽ സൗകര്യ ഐഡന്റിഫയറുകൾ
- **ബന്ധപ്പെടൽ** — ഫോൺ, വിലാസം, അടിയന്തര ബന്ധങ്ങൾ
- **അഡ്മിനിസ്ട്രേറ്റീവ് സന്ദർഭം** — രജിസ്ട്രേഷൻ സ്റ്റാറ്റസ്, സൗകര്യ ബന്ധം, ഡിപ്ലോയ്മെന്റ് ടാഗുകൾ
- **ക്ലിനിക്കൽ ലിങ്കേജ്** — എൻകൗണ്ടറുകൾ, അവസ്ഥകൾ, അലർജികൾ, മറ്റ് റിസോഴ്സുകളിലേക്കുള്ള റഫറൻസുകൾ

രോഗി ഒരു സന്ദർശനത്തിന് തുല്യമല്ല. ഒരു രോഗിക്ക് കാലക്രമേണ പല എൻകൗണ്ടറുകൾ ഉണ്ടാകാം; രോഗി രേഖ ആ ചരിത്രം സുസംഗതമാക്കുന്ന കണ്ടെയ്‌നറാണ്.

## ഐഡന്റിഫയറുകൾ

നിങ്ങളുടെ ഡിപ്ലോയ്മെന്റ് എങ്ങനെ കോൺഫിഗർ ചെയ്തിരിക്കുന്നു എന്നതിനനുസരിച്ച് ഒന്നോ അതിലധികമോ ഐഡന്റിഫയറുകൾ ഉപയോഗിച്ച് രോഗികളെ കണ്ടെത്താം:

| ഐഡന്റിഫയർ തരം | സാധാരണ ഉപയോഗം |
| --- | --- |
| സൗകര്യ MRN | ആശുപത്രി/ക്ലിനിക്കിനുള്ളിൽ പ്രാഥമിക കീ |
| ദേശീയ ആരോഗ്യ ID | ABDM, ABHA, മറ്റ് ദേശീയ റെയിലുകൾ |
| പ്രോഗ്രാം ID | പൊതുജനാരോഗ്യ/കാമ്പെയ്ൻ റെജിസ്ട്രികൾ |

അഡ്മിനിസ്ട്രേറ്റർ നിർവചിക്കുന്ന പരിധിയിൽ ഐഡന്റിഫയറുകൾ അദ്വിതീയമായിരിക്കണം. രജിസ്ട്രേഷനിൽ ഡ്യൂപ്ലിക്കേറ്റ് കണ്ടെത്തൽ ഈ ഫീൽഡുകൾ ഉപയോഗിക്കുന്നു.

## ജീവിതചക്രം

```
രജിസ്റ്റർ → സജീവം → (ഓപ്ഷണൽ) നിഷ്ക്രിയ / മരിച്ചു
```

- **രജിസ്റ്റർ** — കുറഞ്ഞത് ഡെമോഗ്രാഫിക്സ് ഉപയോഗിച്ച് രോഗി രേഖ സൃഷ്ടിക്കുന്നു
- **സജീവം** — എൻകൗണ്ടറുകൾക്കും ഓർഡറുകൾക്കും ഉപയോഗിക്കുന്നു
- **നിഷ്ക്രിയ** — ചരിത്രത്തിനായി സൂക്ഷിക്കുന്നു, പതിവ് തിരയലിൽ മറയ്ക്കുന്നു (കോൺഫിഗർ ചെയ്യാവുന്നത്)
- **മരിച്ചു** — അഡ്മിനിസ്ട്രേറ്റീവ് അവസാനം; ക്ലിനിക്കൽ ചരിത്രം ഓഡിറ്റ് ചെയ്യാവുന്നതാണ്

## അനുമതികൾ

രോഗി ഡെമോഗ്രാഫിക്സ് കാണാനും എഡിറ്റ് ചെയ്യാനും ആർക്കാകും എന്നത് റോൾ അടിസ്ഥാന ആക്സസ് നിയന്ത്രിക്കുന്നു. ക്ലിനിക്കൽ സ്റ്റാഫിന് സാധാരണയായി പൂർണ്ണ രേഖകൾ വായിക്കാം; രജിസ്ട്രേഷൻ സ്റ്റാഫിന് സൃഷ്ടിക്കാനും അപ്ഡേറ്റ് ചെയ്യാനും കഴിയും; ചില ഫീൽഡുകൾ സൗകര്യമോ വാർഡോ പ്രകാരം നിയന്ത്രിക്കപ്പെട്ടേക്കാം.
